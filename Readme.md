<h1>Media Nest</h1>

<p>
Media Nest is a desktop application made for browsing, organizing, tagging, and managing large media collections. It combines a file browser, database-powered search system, media player, thumbnail gallery, and tagging workspace into a single interface designed for fast navigation and large libraries.
</p>

</td>

---

<h2>Introduction</h2>

<p>
Media Nest was created for users who manage large collections of images, videos, GIFs, and downloaded media archives. The application supports both normal folder browsing and database-based media management. Users can open folders directly from their computer or connect a database generated by Kemono Downloader using the <code>library.db</code> file.
</p>

<p>
The application is designed to keep media collections easy to browse even when libraries become extremely large. Instead of navigating thousands of files manually through Windows Explorer, Media Nest organizes everything into searchable trees, thumbnail galleries, and media workspaces that make navigation significantly faster and cleaner.
</p>

<h2>Main Toolbar</h2>

<p>
At the top of the application there are four main buttons that control the core workflow of the application.
</p>

<p>
The <strong>Open Folder</strong> button allows users to manually browse any folder from their computer. This mode does not require a database and works instantly for simple browsing sessions. Once a folder is opened, Media Nest scans the contents and displays them inside the tree view and gallery interface.
</p>

<p>
The <strong>Load DB</strong> button is used for database-based browsing. Media Nest can directly use the <code>library.db</code> database generated by Kemono Downloader. This database becomes available when downloading media from supported sources such as Rule34 and Gelbooru. Once the appdata folder is connected and the database is loaded, users can search media using tags instead of filenames.
</p>

<p>
The <strong>Settings</strong> button opens the full settings dialog where users can configure database paths, interface scaling, tagging tools, duplicate detection, and cloud tagging features.
</p>

<p>
The final button opens the media player in a separate detached window outside the main application. This allows users to keep watching or previewing media while continuing to browse files inside the main window.
</p>

---

<h2>Folder Browsing</h2>

<p>
Media Nest can work like a normal media browser without requiring any database setup. Using the Open Folder feature, users can select any folder from their computer and instantly browse its contents inside the application.
</p>

<p>
The folder structure appears inside the tree view exactly as it exists on disk. Selecting folders automatically updates the gallery view with thumbnails and previews for the files inside that folder. This mode is useful for users who simply want a cleaner and faster way to browse local media collections.
</p>

<p>
Search behavior also changes in this mode. Instead of searching tags from a database, the search bar works using filenames and folder names.
</p>

---

<h2>Database Integration</h2>

<p>
One of the main features of Media Nest is its integration with Kemono Downloader databases. When users download media from supported sources such as Rule34 or Gelbooru using Kemono Downloader, a ready-to-use <code>library.db</code> file is generated automatically.
</p>

<p>
Media Nest can directly connect to this database by selecting the Kemono Downloader appdata folder from the settings window. Once connected, pressing the Load DB button activates database mode and enables tag searching across the entire media library.
</p>

<p>
Instead of searching files manually by folder names, users can search using character names, creators, or custom tags stored inside the database. This makes browsing very large collections significantly faster and more organized.
</p>

<p>
An example database path may look like this:
</p>

<pre><code>E:\Kemono-Downloader\appdata
</code></pre>

---

<h2>Search System</h2>

<p>
Below the toolbar is the main search input used throughout the application. The behavior of the search system depends on whether the user is browsing folders normally or using a loaded database.
</p>

<p>
When database mode is active, the search bar searches tags stored inside the <code>library.db</code> database. Searching instantly updates the gallery with matching images and videos related to the entered tags.
</p>

<p>
When browsing folders manually, the search system instead works using filenames and folder names. This allows Media Nest to remain useful even without a database connection.
</p>

---

<h2>Tree View</h2>

<p>
The left side of the application contains the tree navigation area. This section displays the media library in a clean folder-style hierarchy that makes large collections easier to navigate.
</p>

<p>
Creators, character folders, and media files are organized visually in a structure similar to a normal file explorer, but optimized for large media collections.
</p>

<pre><code>├── creator
│   ├── character name
│   │   ├── file.png
│   │   └── 17384.png
</code></pre>

<p>
Selecting folders or files from the tree immediately updates the media preview and gallery areas.
</p>

---

<h2>Media Player</h2>

<p>
Media Nest includes a built-in media player capable of previewing multiple media formats directly inside the application. The player supports standard images, animated GIFs, videos, and long vertical strip images such as manhwa pages.
</p>

<p>
The player is designed for quick previewing and navigation without needing external applications. Users can play videos, pause playback, control volume, seek through timelines, loop videos, enter fullscreen mode, and quickly skip forward or backward.
</p>

<p>
The media player also supports next and previous navigation, allowing users to browse through large media sets continuously without leaving the preview interface.
</p>

<p>
Long strip images are displayed in a scrollable format so users can comfortably read webtoon or manga-style content directly inside the application.
</p>

---

<h2>Grid Gallery</h2>

<p>
Next to the media player is the grid gallery area. This section displays thumbnails for images and videos in a visual grid layout designed for fast browsing.
</p>

<p>
When searching tags using the database system, the gallery automatically populates with matching media related to the search query. Video files display generated thumbnails alongside image previews so different media types can be browsed together in a single view.
</p>

<p>
When browsing local folders, selecting folders from the tree view updates the gallery with files from the selected location.
</p>

<p>
Users can also control the thumbnail size inside the gallery. The gallery supports large, medium, and small thumbnail modes. Smaller thumbnails allow more files to appear on screen at once, while larger thumbnails provide better preview visibility.
</p>

---

<h2>Settings</h2>

<p>
The settings dialog contains several sections used to configure Media Nest and customize the user experience.
</p>

<h3>Database</h3>

<p>
The Database tab allows users to select the location of the Kemono Downloader appdata folder that contains the <code>library.db</code> file used for tag searching and media indexing.
</p>

<h3>Interface</h3>

<p>
The Interface tab allows users to change the overall application scaling. This is useful for different monitor sizes and resolutions. Users can choose between multiple scaling presets ranging from 50% to 200%.
</p>

<h3>Image Dedup</h3>

<p>
The image duplicate detection tools help users locate and manage repeated image files inside large media libraries.
</p>

<h3>Video Dedup</h3>

<p>
The video duplicate detection tools are designed to detect duplicate or repeated video files and help reduce unnecessary storage usage.
</p>

---

<h2>Tag Manager</h2>

<p>
The Tag Manager is one of the most advanced parts of Media Nest. It provides a complete workspace for organizing, tagging, importing, and managing media files inside the database.
</p>

<p>
Inside the Tag Manager, the interface is divided into four connected sections that work together during the tagging workflow.
</p>

<h3>Tagless Inbox Queue</h3>

<p>
The Tagless Inbox Queue contains media files that currently do not have any tags assigned inside the database. This workspace is designed to help users process and organize untagged files quickly.
</p>

<p>
Users can import additional external folders into the database directly from this section. Imported files become available for tagging and organization without needing manual database editing.
</p>

<p>
The queue also includes the cloud matching system. This feature allows Media Nest users to share tags anonymously using file hashes.
</p>

<p>
If one user manually tags a file and enables anonymous sharing, the file hash and associated tags are uploaded to the cloud database. Later, if another user downloads the exact same file, Media Nest can automatically retrieve the matching tags when the user presses the Find Matches in Cloud button.
</p>

<p>
This system helps reduce repetitive manual tagging across users while improving database consistency.
</p>

<h3>Active Tags</h3>

<p>
The Active Tags section displays all tags assigned to the currently selected media file. Users can manually add new tags, remove unwanted tags, and organize metadata directly from this workspace.
</p>

<p>
Media Nest also includes an integrated AI tagging system. When the AI Tag button is pressed, the application scans the selected image and attempts to detect recognizable content such as characters, visual elements, and general descriptive tags.
</p>

<p>
In many cases the AI system can automatically recognize known characters and assign matching tags without requiring manual input. If any tags are incorrect or missing, users can still modify everything manually.
</p>

<p>
The section also contains tools for deleting selected tags when unwanted metadata needs to be removed.
</p>

<p>
The Media Workspace displays a live preview of the currently selected file from the Tagless Inbox Queue. This allows users to visually inspect media while adding or editing tags.
</p>

<p>
The preview system is designed to make large tagging sessions faster and more comfortable by keeping the image, metadata tools, and tagging controls together inside a single workspace.

<h3>Log Area</h3>

<p>
Below the Media Workspace is the log area. This section provides real-time information about application activity and background operations.
</p>

<p>
The log system displays updates related to database loading, imports, cloud matching, AI tagging, duplicate scanning, and errors. This helps users monitor exactly what the application is currently doing.
</p>

<h2>Supported Media Formats</h2>

<p>
Media Nest supports common image and video formats including PNG, JPG, JPEG, GIF, WEBP, MP4, and WebM files. The application also supports long vertical strip images commonly used for webtoons, comics, and manhwa content.
</p>
